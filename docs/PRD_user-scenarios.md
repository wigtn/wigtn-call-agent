# PRD: WIGVO User Scenarios

> **Project**: WIGVO — AI 전화 대행 플랫폼
> **Version**: 1.0
> **Last Updated**: 2026-02-06

---

## 1. Product Overview

### 1.1 한 줄 요약

**"전화가 유일한 관문인 서비스를, AI가 대신 뚫어주는 음성 비서 플랫폼"**

### 1.2 Problem Statement

| 문제 | 현재 상태 | WIGVO 해결책 |
|------|----------|-------------|
| **콜 포비아** | 전화 통화를 두려워하는 MZ세대 증가 | AI가 대신 전화 |
| **시간 낭비** | 통화 대기, 연결 실패, 재시도 | 비동기로 결과만 확인 |
| **언어 장벽** | 외국인의 한국 로컬 서비스 접근 어려움 | 다국어 입력 → 한국어 통화 |
| **접근성** | 청각 장애인의 전화 서비스 이용 불가 | 텍스트 → 음성 → 텍스트 |
| **허위매물** | 부동산 앱에서 본 매물 확인하려면 결국 전화 | AI가 즉시 확인 |

### 1.3 Target Users

| 페르소나 | 특징 | 주 사용 시나리오 |
|---------|------|----------------|
| **콜포비아 MZ** | 20-30대, 전화 통화 기피 | 예약, 문의 전반 |
| **바쁜 직장인** | 업무 중 전화 어려움 | 점심시간 예약, AS 접수 |
| **외국인 거주자** | 한국어 통화 어려움 | 병원, 관공서, 배달 |
| **청각 장애인** | 음성 통화 불가 | 모든 전화 기반 서비스 |
| **부동산 탐색자** | 매물 확인에 전화 필수 | 허위매물 필터링 |

## 3. Core User Scenarios

### 3.1 Scenario A: 부동산 매물 확인

> **페르소나**: 김서연 (28세, 직장인, 이사 준비 중)
> **Pain Point**: 직방에서 좋은 매물을 찾았는데, 허위매물인지 확인하려면 전화해야 함

#### User Story

```
AS A 부동산 매물 탐색자
I WANT TO AI가 중개사에게 전화해서 매물 상태를 확인해주길
SO THAT 직접 전화하지 않고도 허위매물을 걸러낼 수 있다
```

#### Scenario Flow

| Step | User Action | System Response | Screen |
|------|-------------|-----------------|--------|
| 1 | 앱 실행 | 로그인 화면 표시 | `/login` |
| 2 | Google 로그인 | OAuth 인증 → 메인 화면 | `/` |
| 3 | 요청 입력: "직방에서 본 강남역 OO빌라 201호 아직 있는지 확인해줘" | - | `/` |
| 4 | 전화번호 입력: "02-1234-5678" | - | `/` |
| 5 | [전화 부탁하기] 클릭 | GPT-4가 파싱 → 확인 화면 | `/confirm/[id]` |
| 6 | 파싱 결과 확인 | 대상: OO공인중개사, 매물: OO빌라 201호, 용건: 매물 확인 | `/confirm/[id]` |
| 7 | [전화 걸기] 클릭 | AI가 중개사에 전화 시작 | `/calling/[id]` |
| 8 | 대기 (30초~2분) | 통화 중 애니메이션 + 상태 표시 | `/calling/[id]` |
| 9 | 결과 확인 | "해당 매물은 계약 가능하며, 오후 6시 이후 방문 가능합니다" | `/result/[id]` |
| 10 | [기록 보기] 클릭 | 통화 기록 목록 | `/history` |

#### AI Conversation Script (예상)

```
AI: 안녕하세요, 앱에서 OO빌라 201호 매물 보고 연락드렸는데요,
    아직 계약 가능한가요?

중개사: 네, 아직 나와있어요.

AI: 혹시 오늘 방문 가능할까요?

중개사: 오늘은 좀 어렵고, 내일 오후 6시 이후면 가능해요.

AI: 네, 알겠습니다. 그럼 내일 6시 이후에 방문하겠습니다.
    감사합니다.

[통화 종료]
```

#### Success Criteria

- [ ] 매물 상태 (계약 가능 여부) 확인
- [ ] 방문 가능 시간 파악
- [ ] 30초~2분 내 결과 수신

### 3.2 Scenario B: 미용실 예약

> **페르소나**: 이준호 (25세, 대학원생, 콜포비아)
> **Pain Point**: 단골 미용실이 네이버 예약이 안 되고 전화만 받음

#### User Story

```
AS A 콜포비아 사용자
I WANT TO 텍스트로 예약 요청만 하면 AI가 전화로 예약해주길
SO THAT 전화 통화 스트레스 없이 예약할 수 있다
```

#### Scenario Flow

| Step | User Action | System Response | Screen |
|------|-------------|-----------------|--------|
| 1 | 앱 실행, 로그인 상태 | 메인 화면 | `/` |
| 2 | 요청 입력: "내일 오후 3시에 OO미용실 남자 커트 예약해줘" | - | `/` |
| 3 | 전화번호 입력: "010-1234-5678" | - | `/` |
| 4 | [전화 부탁하기] 클릭 | 파싱: 장소=OO미용실, 날짜=내일, 시간=15:00, 서비스=남자커트 | `/confirm/[id]` |
| 5 | 파싱 결과 확인 및 수정 (필요시) | - | `/confirm/[id]` |
| 6 | [전화 걸기] 클릭 | AI 통화 시작 | `/calling/[id]` |
| 7 | 대기 | "AI가 OO미용실에 전화 중입니다..." | `/calling/[id]` |
| 8 | 결과 확인 | "예약 완료: 내일 15:00 남자커트" | `/result/[id]` |

#### AI Conversation Script (예상)

```
AI: 안녕하세요, 예약 문의 드립니다.
    내일 오후 3시에 남자 커트 예약하고 싶은데 가능할까요?

미용실: 내일 3시요? 잠시만요... 3시는 예약이 차있어서요,
        3시 반은 어떠세요?

AI: 네, 3시 반도 괜찮습니다.
    그럼 내일 3시 반에 남자 커트로 예약 부탁드립니다.

미용실: 네, 성함이 어떻게 되세요?

AI: 이준호입니다.

미용실: 네, 이준호 고객님 내일 3시 반 남자 커트로 예약됐습니다.

AI: 감사합니다. 좋은 하루 되세요.

[통화 종료]
```

#### Success Criteria

- [ ] 예약 성공 여부 확인
- [ ] 최종 예약 시간 (변경된 경우 반영)
- [ ] 예약자 이름 전달

### 3.3 Scenario C: 가전 AS 접수

> **페르소나**: 박민지 (35세, 워킹맘, 시간 부족)
> **Pain Point**: 냉장고 고장났는데 AS 센터 전화하면 대기 시간이 너무 김

#### User Story

```
AS A 바쁜 직장인
I WANT TO AS 접수를 텍스트로 요청하고 결과만 받길
SO THAT 업무 중에도 AS 접수를 할 수 있다
```

#### Scenario Flow

| Step | User Action | System Response | Screen |
|------|-------------|-----------------|--------|
| 1 | 요청 입력: "삼성 냉장고 AS 접수해줘. 냉동실이 안 얼어" | - | `/` |
| 2 | 전화번호 입력: "1588-3366" (삼성전자 서비스) | - | `/` |
| 3 | 파싱 확인 | 용건: AS 접수, 제품: 냉장고, 증상: 냉동실 불량 | `/confirm/[id]` |
| 4 | [전화 걸기] | AI가 AS 센터에 전화 | `/calling/[id]` |
| 5 | 결과 확인 | "AS 접수 완료: 2/8(토) 오전 10-12시 방문 예정" | `/result/[id]` |

#### AI Conversation Script (예상)

```
[ARS 안내 후 상담원 연결]

AI: 안녕하세요, 냉장고 AS 접수하려고 전화드렸습니다.

상담원: 네, 어떤 증상이신가요?

AI: 냉동실이 안 얼어요. 냉장실은 괜찮은데 냉동실만 문제입니다.

상담원: 모델명 확인 가능하신가요?

AI: 잠시만요, 확인이 어려운 상황입니다.
    기사님 방문 시 확인 부탁드려도 될까요?

상담원: 네, 그러면 방문 일정 잡아드릴게요.
        2월 8일 토요일 오전 10시에서 12시 사이 가능하신가요?

AI: 네, 괜찮습니다.

상담원: 주소 확인 부탁드립니다.

AI: 서울시 강남구 테헤란로 123입니다.

상담원: 네, 접수 완료됐습니다.

AI: 감사합니다.

[통화 종료]
```

#### Success Criteria

- [ ] AS 접수 완료
- [ ] 방문 일정 확정
- [ ] 주소 전달 (Dynamic Variable로 전달)

### 3.4 Scenario D: 동네 맛집 예약

> **페르소나**: 최영수 (40세, 회사원, 회식 담당)
> **Pain Point**: 단체 예약은 전화로만 가능한 노포 맛집들

#### User Story

```
AS A 회식 담당자
I WANT TO 단체 예약을 AI가 대신 전화로 잡아주길
SO THAT 업무 시간에 전화하기 어려운 상황에서도 예약할 수 있다
```

#### Scenario Flow

| Step | User Action | System Response |
|------|-------------|-----------------|
| 1 | 요청: "오늘 저녁 7시에 OO삼겹살 8명 예약해줘. 룸으로 부탁해" | - |
| 2 | 전화번호: "02-555-1234" | - |
| 3 | 파싱 확인 | 장소=OO삼겹살, 날짜=오늘, 시간=19:00, 인원=8명, 요청=룸 |
| 4 | 결과 | "예약 완료: 오늘 19:00, 8명, 룸 배정됨" 또는 "룸 불가, 홀 자리만 가능" |

#### Success Criteria

- [ ] 단체 예약 가능 여부 확인
- [ ] 룸/홀 등 좌석 요청 전달
- [ ] 최종 예약 확정

### 3.5 Scenario E: 병원 예약 (외국인)

> **페르소나**: John Smith (32세, 미국인, 한국 거주 2년차)
> **Pain Point**: 한국어 전화 통화가 어려워서 병원 예약이 힘듦

#### User Story

```
AS A 한국 거주 외국인
I WANT TO 영어로 요청하면 AI가 한국어로 전화해주길
SO THAT 언어 장벽 없이 병원 예약을 할 수 있다
```

#### Scenario Flow

| Step | User Action | System Response |
|------|-------------|-----------------|
| 1 | 요청 (영어): "Book a dentist appointment for tomorrow afternoon at OO Dental Clinic" | - |
| 2 | 전화번호: "02-333-4444" | - |
| 3 | 파싱 확인 (영→한 번역) | 장소=OO치과, 날짜=내일, 시간=오후, 용건=진료 예약 |
| 4 | AI 통화 (한국어) | AI가 한국어로 치과에 전화 |
| 5 | 결과 (영어) | "Appointment confirmed: Tomorrow 2:30 PM at OO Dental" |

#### Success Criteria

- [ ] 영어 입력 → 한국어 파싱 → 한국어 통화
- [ ] 결과를 영어로 요약 (향후 기능)
- [ ] 의료 용어 정확한 전달

## 5. Data Requirements

### 5.1 Dynamic Variables (AI에게 전달)

| Variable | Description | Example | Source |
|----------|-------------|---------|--------|
| `target_name` | 통화 대상 이름 | "OO미용실" | GPT-4 파싱 |
| `date` | 예약 날짜 | "내일", "2026-02-07" | GPT-4 파싱 |
| `time` | 예약 시간 | "오후 3시", "15:00" | GPT-4 파싱 |
| `service` | 서비스/용건 | "남자 커트", "매물 확인" | GPT-4 파싱 |

| `address` | 주소 (AS용) | "서울시 강남구..." | 사용자 입력 |
| `party_size` | 인원수 | "8명" | GPT-4 파싱 |
| `special_request` | 특별 요청 | "룸으로 부탁해요" | GPT-4 파싱 |

## 6. Edge Cases & Error Handling

### 6.1 통화 실패 케이스

| Case | Detection | User Message | Action |
|------|-----------|--------------|--------|
| 부재중 | 30초 이상 무응답 | "전화를 받지 않았습니다" | 재시도 버튼 |
| 통화중 | Busy signal | "통화중입니다. 잠시 후 다시 시도해주세요" | 자동 재시도 옵션 |
| 결번 | Invalid number | "존재하지 않는 번호입니다" | 번호 수정 |
| ARS 무한루프 | 3분 초과 | "자동응답 시스템에서 진행이 어렵습니다" | 직접 전화 안내 |
| 예약 거절 | AI 판단 | "요청하신 시간에 예약이 불가능합니다" | 대안 제시 |

### 6.2 파싱 실패 케이스

| Case | Example Input | Handling |
|------|---------------|----------|
| 날짜 불명확 | "이번 주에 예약해줘" | 사용자에게 구체적 날짜 요청 |
| 시간 불명확 | "오후에 예약해줘" | "오후 몇 시쯤 원하시나요?" |
| 장소 불명확 | "미용실 예약해줘" | "어떤 미용실인가요?" |
| 전화번호 없음 | 요청만 입력 | "전화번호를 입력해주세요" |
